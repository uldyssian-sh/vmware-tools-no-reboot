# Contributing to VMware Tools No-Reboot Upgrade

Thank you for your interest in contributing to this project! We welcome contributions from the community.

## How to Contribute

### Reporting Issues
- Use the [GitHub Issues](https://github.com/uldyssian-sh/vmware-tools-no-reboot/issues) page
- Provide detailed information about the issue
- Include PowerShell version, VMware Tools version, and environment details

### Submitting Changes
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Test thoroughly
5. Commit your changes (`git commit -am 'Add new feature'`)
6. Push to the branch (`git push origin feature/your-feature`)
7. Create a Pull Request

## Development Guidelines

### Code Standards
- Follow PowerShell best practices
- Use proper error handling
- Include comprehensive help documentation
- Test with different VMware Tools versions

### Testing
- Test in lab environments before production
- Verify compatibility with different vSphere versions
- Include unit tests where applicable

### Documentation
- Update README.md if needed
- Document any new parameters or features
- Include usage examples

## Code of Conduct
Please be respectful and professional in all interactions.

## Questions?
Feel free to open an issue for questions or discussions.
